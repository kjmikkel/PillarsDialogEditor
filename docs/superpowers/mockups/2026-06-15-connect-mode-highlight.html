Connect mode — distinguishing the "connection source" node from the normal selection

In connect mode, two nodes need distinct highlights at once: the fixed source node (where the new connection will start) and the target candidate (the currently-selected node, which still gets the normal blue selection ring as arrow keys move it). Per the project's non-colour-encoding rule, the source highlight needs a shape/pattern cue, not just a different hue — so it stays legible in Colourblind and High-Contrast palettes too.

Option A — Dashed border + 🔗 badge (top-left)

Mirrors the existing diff-status pattern (coloured border + corner glyph badge), but dashed instead of solid, and placed top-left so it never collides with the diff badge (top-right).
