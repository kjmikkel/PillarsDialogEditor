# Project Autosave / Crash Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A 60-second sidecar autosave (`<project>.dialogproject.autosave`) while the project is dirty, with an offer-to-restore on the next open and deletion on save or deliberate discard.

**Architecture:** Pure `AutosaveRecovery` helper (sidecar path + newer/stale/none check) in `DialogEditor.ViewModels/Services`; `MainWindowViewModel` gains `AutosaveTick()` (reuses `FoldCanvasIntoProject` + `DialogProjectSerializer`), sidecar deletion in `SaveProject`/`SaveProjectAs`/`DiscardAndProceed`, and a restore hook in `LoadProjectAsync` behind an injectable confirm seam; `MainWindow` wires a `DispatcherTimer` and a small `AutosaveRestoreDialog`.

**Tech Stack:** C# / .NET 8, Avalonia `DispatcherTimer`, xUnit. No new dependencies.

## Global Constraints

- **TDD, red first.** Suite runs serially; tests use temp paths + seams, never real app data.
- **Sidecar path rule:** exactly `projectPath + ".autosave"` (single generation).
- **Autosave never interrupts writing:** every failure path is caught and logged (`AppLog.Warn`); no status-bar output on ticks (`AppLog.Info` on success); `IsModified` is never cleared by a tick.
- **Restore semantics:** restored content loads under the real `_projectPath` with `IsModified = true`; the sidecar is kept until the next explicit save (double-crash protection). Discard deletes the sidecar. Stale sidecars (older than the project file) are silently deleted at open.
- **Deliberate-discard deletion happens in `DiscardAndProceed`** (MainWindowViewModel.cs:194) — the single choke point for all dirty-guard discards (app close, New/Open/Close Project, branch switch).
- **Localisation / tooltips / UIA:** dialog strings are resources; buttons carry ToolTip + mirrored HelpText (structural suites enforce).
- **Build/test:** `dotnet test DialogEditor.Tests/DialogEditor.Tests.csproj`.

### Task 1: `AutosaveRecovery` (pure helper)

**Files:**

**Interfaces:**
- `public enum AutosaveState { None, Stale, Newer }`
- `public sealed record AutosaveCheckResult(AutosaveState State, string SidecarPath, DateTime? SidecarTimeUtc);`
- `public static class AutosaveRecovery { public static string SidecarPath(string projectPath); public static AutosaveCheckResult Check(string projectPath); public static void TryDelete(string projectPath); }`
- `Check`: sidecar missing → `None`; sidecar `LastWriteTimeUtc` > project file's → `Newer` (with timestamp); otherwise → `Stale`. Project file missing but sidecar present → `Newer` (recover-only situation). All IO exceptions → `None` + `AppLog.Warn`.
- `TryDelete`: best-effort delete of the sidecar, `AppLog.Warn` on failure, never throws.

- Seam: `public Func<DateTime, Task<bool>>? ConfirmRestoreAutosave { get; set; }` — argument is the sidecar's local timestamp; `true` = restore. **Null seam → treat as discard-the-offer but keep the sidecar** (never destroy recovery data because a dialog wasn't wired; load the real file normally).
- In `LoadProjectAsync(path, offerDeferred)` before reading the project text:

```csharp
        var recovery = AutosaveRecovery.Check(path);
        if (recovery.State == AutosaveState.Stale)
        {
            AutosaveRecovery.TryDelete(path); // save happened through another route
        }
        else if (recovery.State == AutosaveState.Newer && ConfirmRestoreAutosave is not null)
        {
            var restore = await ConfirmRestoreAutosave(recovery.SidecarTimeUtc!.Value.ToLocalTime());
            if (restore)
            {
                try
                {
                    var recovered = DialogProjectSerializer.LoadFromFile(recovery.SidecarPath);
                    FinishLoad(recovered, path);
                    IsModified = true;   // recovered state is unsaved until an explicit save
                    StatusText = Loc.Get("Status_AutosaveRestored");
                    return;              // sidecar is KEPT until that save
                }
                catch (Exception ex)
                {
                    AppLog.Warn($"Autosave restore failed for '{path}': {ex.Message} — loading saved file");
                    AutosaveRecovery.TryDelete(path); // corrupt sidecar: don't re-offer forever
                }
            }
            else
            {
                AutosaveRecovery.TryDelete(path);    // user declined — respect it
            }
        }
```
